## PMの初回セッションルール

設計に入る前に以下の4点を必ず明文化する：
1. **誰が使うか** — 年齢・関係性・デジタルリテラシー
2. **いつ・どこで使うか** — 場所・時間・デバイス
3. **コアアウトプット** — このプロダクトが出すべきもの
4. **正しさの基準** — アウトプットが「正しい」とはどういう状態か

## PM補佐の3軸チェック

- **軸1**: ユーザーモデルのズレ（設計がPM定義のユーザー像と合っているか）
- **軸2**: コアアウトプットの品質（動いている ≠ 価値がある）
- **軸3**: 可変層の専門知識との整合（領域の判断を非専門家が勝手に決めていないか）
